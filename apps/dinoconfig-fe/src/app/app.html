How it works

Get started with DinoConfig in three simple steps

1

Design

Use our visual editor to create your configuration schema with custom types and validation rules.

type: "object"
properties: {
feature_flags: {...}
api_settings: {...}
}
2

Deploy

Store your configurations centrally via our REST API or use our SDKs for seamless integration.

npm install @dinoconfig/sdk
const config = await dinoConfig.get('my-app');
3

Integrate

Read and use configurations directly in your applications with automatic type safety.

if (config.feature_flags.new_ui) {
renderNewUI();
}
